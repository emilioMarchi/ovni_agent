=====================================
DEPLOY LIMPIO ovni-agent → api.ovnistudio.com.ar
=====================================


PREREQUISITOS:
- VPS Ubuntu/Debian en DigitalOcean
- Acceso solo por consola web (sin SSH externo)
- Dominio api.ovnistudio.com.ar apuntando a 64.23.138.249

=====================================
0. INSTALAR DEPENDENCIAS BÁSICAS
============================ =========

# Actualiza paquetes
sudo apt update && sudo apt upgrade -y

# Instala curl y unzip si no están
sudo apt install curl unzip -y

# Instala nginx
sudo apt install nginx -y


# Instala nvm (Node Version Manager)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# Si después de instalar nvm el comando 'nvm' no funciona, ejecuta esto en la terminal:
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh"

# Si sigue sin funcionar, ejecuta:
bash
# y luego repite los comandos anteriores.

# Instala Node.js 18 LTS y npm
nvm install 18
nvm use 18
nvm alias default 18

# Verifica versiones
node -v
npm -v

# Instala pm2 global
npm install -g pm2

=====================================

=====================================
1. DNS — Panel de tu proveedor de dominio
=====================================

Tipo: A
Host: api
Valor: 64.23.138.249
TTL: Auto

Desactiva el proxy (nube gris) en Cloudflare para emitir el SSL.

=====================================
2. SUBIR PROYECTO AL SERVIDOR
=====================================

1. Desde tu PC, crea un .zip de todo el proyecto (sin node_modules ni dist).
2. Sube el .zip al VPS usando el panel de archivos de DigitalOcean o wget/curl si tienes un link.
3. En la consola web:
  mkdir -p /root/ovni_agent
  cd /root/ovni_agent
  unzip /ruta/al/zip
4. Instala dependencias:
  npm install

=====================================
3. CONFIGURAR VARIABLES DE ENTORNO
=====================================

1. Crea el archivo .env:
  nano /root/ovni_agent/.env

2. Ejemplo de contenido:
  PORT=8080
  NODE_ENV=production
  GEMINI_API_KEY=tu_key
  PINECONE_API_KEY=tu_key
  # ... otras variables necesarias

3. Sube agent-firebase-service.json al mismo directorio (/root/ovni_agent).

=====================================
4. BUILD Y PM2
=====================================

cd /root/ovni_agent
npm run build
mkdir -p logs
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
# Ejecuta el comando que te muestre pm2 startup

=====================================
5. NGINX — Configuración inicial para SSL
=====================================

1. Copia nginx/ovni-agent.conf a /etc/nginx/sites-available/ovni-agent.conf
2. Edita ovni-agent.conf y deja SOLO el bloque server { listen 80; ... } (sin SSL, sin 443)
3. Habilita el site:
  sudo ln -s /etc/nginx/sites-available/ovni-agent.conf /etc/nginx/sites-enabled/
  sudo rm -f /etc/nginx/sites-enabled/default
4. Recarga nginx:
  sudo nginx -t
  sudo systemctl reload nginx

=====================================
6. SSL — Emitir certificado con Certbot
=====================================

sudo apt update
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d api.ovnistudio.com.ar
# Sigue los pasos: email, ToS, redirect HTTP→HTTPS (Yes)

Si todo sale bien, certbot agregará el bloque SSL automáticamente.
Si no, edita ovni-agent.conf y agrega el bloque SSL manualmente (puedo darte el ejemplo si lo necesitas).

Vuelve a activar el proxy naranja en Cloudflare si lo usas.

=====================================
7. VERIFICAR Y LOGS
=====================================

curl https://api.ovnistudio.com.ar/health
pm2 logs ovni-agent
sudo tail -f /var/log/nginx/error.log
sudo nginx -t
sudo ufw status

=====================================
8. FIREWALL (si aplica)
=====================================

sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 22/tcp
sudo ufw enable
sudo ufw status

=====================================
9. ACTUALIZAR EN EL FUTURO
=====================================

cd /root/ovni_agent
rm -rf *
# Sube y descomprime el nuevo .zip
npm install
npm run build
pm2 restart ovni-agent
